## الخطوة 1: تدقيق البيانات (أولوية قصوى — Q1)

### 1.1 التدقيق البشري لعينة الجذور الثلاثة
```
المهمة:        تدقيق 357 كلمة عشوائية (±5% خطأ عند 95% ثقة)
الأداة:        Airtable / Google Sheets + 3 مدققين مستقلين
المعيار:       هل تنتمي الكلمة فعلاً لـ ATOM/BULL/TOR حسب تعريف المشروع؟
المدة:         3 أسابيع
التحقق:        Cohen's Kappa ≥ 0.80 بين المدققين
المسؤول:       أنا (عالم البيانات)
```
- **خرّج** عينة طبقية عشوائية من 5,083 كلمة (حسب الجذر)
- **أنشئ** نموذج تدقيق: `word | root_claimed | root_1 | root_2 | root_3 | consensus | confidence`
- **احسب** دقة المصنف الآلي بعد التدقيق

### 1.2 استخراج الترددات النصية الفعلية
```
المهمة:        استبدال تقديرات الخبراء بترددات حقيقية
الأداة:        CODA corpus + Arabic TenTen + Sketch Engine
المدة:         4 أسابيع
التحقق:        مقارنة الترددات الجديدة مع القديمة — هل يتغير r=−0.693؟
المسؤول:       أنا (عالم البيانات)
```
- **استخرج** ترددات الحروف الـ 28 من CODA
- **أعد حساب** correlation (r) بعد البيانات الحقيقية
- **طبّق** تصحيح Bonferroni على جميع الاختبارات

### 1.3 توثيق منهجية AI

## الخطوة 2: المشاريع البرمجية (أولوية عالية — Q1-Q2)

### 2.1 [P1] CCM Algorithm — أقصر طريق للنشر
```
الوصف:        خوارزمية مطابقة أول حرفين ساكنين عبر 9 فئات صوتية
البيانات:     WALS, Lexibank, ASJP (آلاف اللغات)
المنهج:       Python + distance matrix + random baseline
النشر:        Journal of Quantitative Linguistics (short paper)
المدة:        6 أسابيع
خطوات:
  Phase 1:    برمجة الـ 9 فئات الصوتية بناءً على ibdal_rules.json
  Phase 2:    اختبار على 5 أزواج لغوية (عربي-عبراني، عربي-لاتيني...)
  Phase 3:    مقارنة مع Levenshtein distance كـ baseline
  Phase 4:    كتابة الورقة
```

### 2.2 [P2] Self-Correcting Text Structure (Tenen)
```
الوصف:        هل يمكن لخوارزمية استعادة حرف مفقود في النص بناءً على
              التناظر الهندسي لمكعب تينين Base-3؟
البيانات:     Hebrew Torah (first verse), Arabic Bismillah
المنهج:       Python + 3x3x3 cube coordinates + error injection
النشر:        Digital Scholarship in the Humanities
المدة:        4 أسابيع
خطوات:
  Phase 1:    بناء مكعب Base-3 (27 حرفاً = 27 موقعاً)
  Phase 2:    برمجة خوارزمية التناظر
  Phase 3:    اختبار استعادة أحرف عشوائية
  Phase 4:    مقارنة مع random baseline
```

### 2.3 [P3] Semantic Suffix Clustering — أقوى نمط متكرر
```
الوصف:        هل تشكل الكلمات المنتهية بـ -tion/-ble/-tor
              عناقيد دلالية في فضاء تضمين الكلمات؟
البيانات:     5 لغات (EN, ES, IT, FR, PT) من fastText/mBERT
المنهج:       Word Embeddings + t-SNE + silhouette score
النشر:        COLING أو ACL workshop
المدة:        8 أسابيع (أكبر مشروع)
خطوات:
  Phase 1:    استخراج all words ending in -tion/-ble/-tor من 5 قواميس
  Phase 2:    حساب Word Embeddings (mean vector لكل مجموعة)
  Phase 3:    اختبار: هل المسافة بين عناقيد اللواحق أقل من اللواحق العشوائية؟
  Phase 4:    تكرار عبر 5 لغات — هل النمط ثابت؟
  Phase 5:    تحليل silhouette score لكل عنقود
  Phase 6:    كتابة الورقة
```
